**Current order is wrong for agri-ops.** The order must reflect the real daily workflow of a farm manager:

**Correct order (top → bottom):**
1. Головна панель (Dashboard)
2. Поля (Fields)
3. Склад і логістика (Warehouse group — Warehouses, Stock, Movements, Grain storage)

File: most likely `frontend/src/components/Layout/AppLayout.tsx` or a sidebar menu config near it. Change only the group order; do not rename items, do not touch permissions/roles.

### 2. Dashboard redesign — `frontend/src/pages/Dashboard.tsx` + its sub-components

Keep the same 4 KPI tiles, same 3 widgets (alerts, chart, fields+operations lists), but elevate the execution:

**KPI hero row (`KpiHeroRow`)** — 4 cards: area, season expenses, season revenue, season profit + margin.
- Each card: title small uppercase, big number (Geist, tabular-nums), small delta chip with arrow (+5%, −3%) using green/amber tokens — not red for spend increases unless over budget
- Subtle top-left radial glow in the accent color, matching the card's KPI type (area = neutral white, expenses = amber, revenue = green, profit = accent green with glow)
- 1px border in `--surface-stroke`, slight inner highlight on top edge, no heavy drop-shadow

**Period switcher (Day / Week / Month / Season)** — segmented control, top-right of the page header. Already exists — keep it, just polish: active state = filled accent, inactive = ghost. Persist choice in a local Zustand slice or `useState` (do not add a new store).

**Alerts panel** — replace the two red strips with a proper "Потребує уваги" card:
- Sectioned by severity: Critical (red), Warning (amber), Info (muted blue)
- Each row: icon + title + count + chevron link to the relevant page
- Max 5 rows visible, "Показати всі (N)" link below

**Revenue vs Cost chart** — currently just a red line. Needs:
- Two series: expenses (amber, area under) + revenue (green, line on top)
- Month labels on X axis, formatted values on Y (₴ k / ₴ m auto-scale)
- Hover tooltip showing both series and the profit delta
- If project already uses Recharts/Chart.js — use it. If not, use whatever chart lib is already in `package.json`. **Do not add a new one.**

**Fields status + Operations timeline** — two-column grid, equal height. Fields list: chip with crop icon/color, name, area, current stage. Operations timeline: vertical stepper, completed = filled green dot, pending = hollow, overdue = amber pulse.

**Quick actions strip** — four buttons (create operation, add cost, record movement, log salary) — reduce visual weight, make them ghost buttons in a single row, not big tiles.
