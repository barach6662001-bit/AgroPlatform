=== GLOBAL TECHNICAL ASSESSMENT ===
1) Real level now:
- pseudo-enterprise with hidden fragility.
- Broad feature surface exists, but many items are PARTIAL and several are integration-misaligned.

2) Architectural resilience:
- Frontend: medium-low (feature-rich but duplicated modules, inline styling, inconsistent loading/delete patterns).
- Backend: medium (CQRS present, but some controllers bypass abstraction; several “good enough” shortcuts).
- CI: medium-low (tests pass but with significant warnings/noisy async handling; dotnet checks not runnable in this environment).
- Domain model: medium (wide coverage, but quality uneven; some fake-complete stories).
- Analytics layer: medium-low (many charts, weaker data provenance semantics).
- Spatial layer: medium (core pieces present, but several parts synthetic/heuristic).
- Permissions/audit/multi-farm foundations: low-medium (skeleton exists, enterprise claims overstated).

3) Strong vs fragile:
- Strong: breadth of implemented modules, routing coverage, typed frontend API, CQRS structure, SignalR fleet pipeline.
- Fragile: fake/synthetic behavior presented as complete capability, duplication, weak source-of-truth for permissions, partial offline/push/multi-farm.

4) Where it will break at scale:
- More users: policy drift and authorization inconsistencies.
- More data: chart pages and ad-hoc queries likely degrade; limited completeness metadata.
- More farms: tenant UX and mapping model insufficient.
- More roles: static hardcoded matrix will explode operationally.
- More analytics: duplicated FE modules diverge.
- Mobile/offline: conflict handling insufficient.
- Integrations: webhook contracts too soft (unknown devices, silent partial failures).

5) Top-10 dangerous systemic problems:
1. Task ledger/process drift (041 mismatch) undermines delivery trust.
2. “Done” features often partial/synthetic.
3. Role/permission model not canonical/persisted.
4. Multi-farm UI not wired (FarmSwitcher dead).
5. Offline replay lacks conflict strategy.
6. Push pipeline incomplete (subscribe-only).
7. Skeleton/loading UX inconsistent.
8. Duplication in analytics/marginality modules.
9. Geofence dedup via string matching.
10. Audit coverage manually allowlisted and not verifiably complete.

6) Top-5 immediate fixes (before new big features):
1. Reopen 046 and fully wire multi-farm UX + user-tenant mapping.
2. Reopen 044 with persistent permission matrix and admin UI.
3. Reopen 043 with real push delivery service.
